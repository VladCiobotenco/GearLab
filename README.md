# Web Techniques Project Documentation: GearLab

## 1. Included Information
The proposed website aims to sell peripheral products for PCs and laptops. Users will have the opportunity to analyze the technical details of the products offered for sale and add their favorite items to the shopping cart. Users can also observe the history of promotions and future upcoming offers to make an informed purchasing decision.

### User Experience:
* **Navigation:** The user can navigate between the available product types: mice, keyboards, headsets, mousepads, and console controllers.
* **Product Details:** For each device, the user can view the description and technical specifications of the product.
* **Interactivity:** If a user likes a product, they can add it to the shopping cart, which updates in real-time. The shopping cart represents a list of products and their total price.
* **Planning:** The user can analyze past offers and future predictions.
* **Branding:** The user can learn about the GearLab store's story and find contact information.

---

## 2. Site Structure
The site is divided into **4 main pages**, accessible from the header:

1.  **Home** – general information about the site's purpose; offers.
2.  **About Us** – the origin and history of the site.
3.  **Contact** – contact methods (address, phone, email).
4.  **Shopping Cart** – the list of products added by the user to the cart.

Additionally, from the **Home** page, the user can access 5 different pages based on the product category:
* `Mice`
* `Keyboards`
* `Headsets`
* `Mousepads`
* `Console Controllers`

> [!NOTE]
> Each page contains products from the selected category, and the user always has the option to return to the Home page.
